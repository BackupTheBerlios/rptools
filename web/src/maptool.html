MapTool

Introduction

One member of our group is in NV (we're in TX), and we sought to solve all of the long-distance role playing hurdles. Voice, text, die rolling, and the battle map. There are tools available for many of these problems.

For map sharing, we started with Klooge Werks. A very fine piece of code that is reasonably priced. However our GM's "I'll describe, you draw" style was not very well supported by KW's drawing tools nor is it scalable (the GM is the only one that can draw/move things). What we really wanted was something much simpler which actually just made small improvements over the battlemap, minis, and wet-erase markers. KW does a whole lot more than what we were looking for.

Features

  • client/server
  • standalone server operation supported
  • load images for map backgrounds
  • load directories of images for marker sources
  • draw on a drawing layer
  • move markers around on the map
  • pan and zoom client features
  • Java 1.5 (WebStart supported)
